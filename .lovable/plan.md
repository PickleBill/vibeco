

## Plan: AI-Generated Landing Page Preview in Simulator

### The Idea

After the simulator generates the final report with a `lovable_prompt`, we use an AI model to generate actual HTML/CSS for a lightweight landing page and render it live in the report. The user sees a real, interactive preview of what their product landing page could look like — not just a screenshot image, but actual rendered HTML in a sandboxed iframe.

### Why HTML Generation > Image Generation

- An image of a landing page is static and can't be interacted with
- Generated HTML is inspectable, feels real, and demonstrates VibeCo's capability
- Users can view source, see it's real code, and feel the "wow"
- We already have the `lovable_prompt` which describes exactly what to build — we just need to ask a model to turn it into HTML

### Architecture

```text
┌─────────────────────┐
│  simulate-idea      │  (already generates lovable_prompt on final round)
│  edge function      │
└────────┬────────────┘
         │ lovable_prompt
         ▼
┌─────────────────────┐
│  generate-landing   │  NEW edge function
│  page               │  Takes lovable_prompt → returns single-file HTML
└────────┬────────────┘
         │ html string
         ▼
┌─────────────────────┐
│  FinalReport.tsx     │  Renders HTML in sandboxed iframe via srcdoc
│  (new section)       │  + "View Full Page" expands to full-screen modal
└─────────────────────┘
```

### New Edge Function: `generate-landing-page`

- **Input:** `{ prompt: string }` (the lovable_prompt)
- **Model:** `google/gemini-2.5-flash` (fast, good at code generation, cheap)
- **System prompt:** "You are a frontend developer. Generate a single self-contained HTML file with inline CSS and no external dependencies. Create a beautiful, modern landing page based on the following product brief. Use a dark theme with teal/cyan accents. Include: hero section, features grid, pricing, testimonials placeholder, and CTA. Make it responsive. Return ONLY the HTML — no markdown, no explanation."
- **Output:** `{ html: string }` — the raw HTML string
- **Stored in:** `simulator_captures.landing_page_html` (new text column, nullable)

### Database Change

Add one column:
```sql
ALTER TABLE public.simulator_captures ADD COLUMN landing_page_html text;
```

### FinalReport.tsx Changes

After the unlocked report sections and before the Lovable Prompt section:

1. **"Generate Preview" button** — appears once the report is unlocked and `lovablePrompt` exists
2. On click, calls the edge function, shows a loading state ("Building your landing page...")
3. Once HTML is returned, renders in a **sandboxed iframe** using `srcdoc` attribute
4. iframe is displayed in a styled container (phone/desktop frame mockup) with a border
5. **"View Full Screen"** button opens the HTML in a modal/dialog at full viewport
6. The generated HTML is saved to the database for persistence

### SimulatorShell.tsx Changes

- After the final report generates successfully and `lovablePrompt` is available, automatically trigger the landing page generation in the background (non-blocking)
- Pass `landingPageHtml` as a new prop to `FinalReport`
- If it was previously generated (loaded from DB), pass it directly without re-generating

### UX Flow

1. User completes simulation → sees final report (as today)
2. Below the report sections, a new prominent card appears: **"See Your Landing Page"** with a sparkle animation
3. If not yet generated: clicking triggers generation with a 5-8 second loading animation
4. If already generated (from DB or just created): shows the iframe immediately
5. Below the iframe: "This is a real, working preview generated by AI. Ready to make it real?" → CTA to contact
