

## Plan: Interactive Report Deep-Dive + Cleanup Landing Page Remnants

### What We're Building

Two features that make the report feel alive and actionable:

**1. "Deep Dive" on any report section** — Each of the 7 report sections (Problem, Target Customer, Features, etc.) gets a subtle expand button. Clicking it calls the AI to generate a richer, more detailed analysis of just that section, rendered inline below the original text. Think of it as "click to zoom in" on any part of the analysis.

**2. Strip the landing page generation feature** — Remove the iframe preview, the `landingPageHtml` state, and the `onGenerateLandingPage` props. Keep the Lovable Prompt copy section (that's the real value).

### Deep Dive UX

- Each section card gets a small **"Deep dive ↓"** button (ghost style, bottom-right)
- Clicking it shows a loading shimmer, then reveals an expanded panel with:
  - 3-4 bullet points of deeper analysis (specific competitors named, market size estimates, risk factors, implementation considerations)
  - A subtle border-left accent to distinguish it from the original text
- Only one section can be deep-diving at a time (clicking another collapses the first)
- The deep dive content is generated by calling the existing `simulate-idea` edge function with a new `type: "deep_dive"` that takes the section key + existing brief context

### Architecture

```text
User clicks "Deep dive" on "Industry & Competitors"
         │
         ▼
simulate-idea edge function (type: "deep_dive")
  Input: { type: "deep_dive", section: "industry_trends", brief: {...}, idea: "..." }
  Output: { deep_dive: "Detailed markdown analysis..." }
         │
         ▼
Rendered inline below the section with markdown support
```
